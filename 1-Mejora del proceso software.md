# Mejora del proceso software

## Objetivos

Para saber si se puede mejorar un proceso _software_ es necesario tener en cuenta los objetivos que se quieren alcanzar y lo lejos que están:

- Desde el punto de vista de la gestión del proyecto: Cumplir con el esfuerzo, tiempo y coste estimados.
- Desde el punto de vista del producto: Lograr una alta calidad.

Los anteriores aspectos deben poder medirse para su cuantificación y comparación:

- Esfuerzo: `unidad de tiempo * hombre`.
- Tiempo: `unidad de tiempo`.
- Coste: `unidad monetaria`.

### Estimación de la calidad

La calidad es un aspecto subjetivo y los parámetros para medirla tienen que ser definido para cada empresa o producto.

Existen limitaciones al establecer los parámetros. Deben ser:

- Establecidos con anticipación desde el punto de aceptación elegido.
- Cuantificables.
- Verificables.

## Mejoras

Las mejoras deben centrarse en 3 aspectos interrelacionados:

- Desarrollo:
  - Elección de un ciclo de vida adecuado.
- Gestión:
  - Aplicar aproximaciones sistemáticas en planificación, seguimiento y cierre.
  - Mejorar la estimación, el seguimiento y el control de esfuerzo, tiempo y coste.
- Calidad:
  - Realizar el producto con calidad desde el comienzo.
  - Utilizar las pruebas como un complemento.
